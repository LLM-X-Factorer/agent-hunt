# 点头人工智能系统班 V7.0 —— AI 应用工程师实战线

---

## 定位

系统班第四条独立课程线，与影像/时序/AI+数据三个实验室并行。面向有就业需求的学员，以国内 AI 就业市场真实岗位需求为导向，系统覆盖 LLM、RAG、Agent、Prompt Engineering 四大高频技能，最终交付一个可运行的「智能知识库 Agent」系统。

**前置要求：** 完成 Phase 1 新手通识（Python 编程基础、机器学习、深度学习概念）

**核心交付物：** 一个从零构建的、可部署运行的智能知识库 Agent 系统（带知识库检索 + 多步任务执行能力）

---

## 设计理念

### 为什么做这条线

国内 AI 就业市场数据（基于 1200+ 条国内岗位 JD 分析）显示：

| 技能 | 岗位出现频次 | 占比 |
|------|------------|------|
| 大语言模型（LLM） | 174 | 14.5% |
| Agent 架构 | 69 | 5.8% |
| Python | 38 | 3.2% |
| RAG | 22 | 1.8% |
| Prompt Engineering | 20 | 1.7% |

这四项技能构成了国内 AI 岗位最核心的技能集。本课程围绕一个完整项目，将这四项技能全部覆盖——学员完成课程后，面对招聘市场上的主流 AI 岗位 JD，会发现「这些关键词我都做过」。

### 一个项目，两阶段递进

课程围绕同一个系统从简单到复杂自然演进：

```
前半段：RAG 智能知识库 —— 学员建立 LLM 应用开发的基本能力
    ↓（在知识库基础上扩展）
后半段：知识库 Agent —— 加入工具调用、多步推理、工作流编排
    ↓
最终交付：一个带知识库的智能 Agent 系统
```

不是两个割裂的项目，而是同一个系统的持续迭代。这本身也模拟了真实工作中产品演进的过程。

### Vibe Coding 作为工作方式

全程鼓励使用 AI 辅助编程（Cursor/Claude Code 等）。课程不考核「能不能手写代码」，而是考核「能不能用 AI 把系统跑通、能不能读懂和解释 AI 生成的代码、能不能发现和修复问题」。这本身就是当前 AI 岗位最核心的工作方式。

## 课程内容

### 模块 1：LLM 应用开发基础（4 课时，直播）

从 Phase 1 的「理解 AI 原理」过渡到「用 LLM 做应用」，建立开发心智模型。

| 课时数 | 课题描述 | 课程形式 |
|-------|--------|--------|
| 1课时 | **AI 就业市场全景 + LLM API 调用实践** | 直播 |
| | 前 20 分钟：基于 1200+ 条国内真实 JD 数据，展示 AI 岗位分布、薪资水平、高频技能需求，建立课程与市场的对应关系；后 40 分钟：国内主流 API（DeepSeek/通义千问）；请求/响应结构；对话管理；Token 与上下文窗口；流式输出 | |
| 1课时 | **Prompt Engineering 系统化方法** | 直播 |
| | 系统提示词设计；角色设定；Few-shot/Chain-of-Thought；结构化输出（JSON mode）；Prompt 评估与迭代方法 | |
| 1课时 | **向量嵌入与语义检索** | 直播 |
| | 文本向量化原理；Embedding 模型选择；向量数据库（Chroma/Milvus）；相似度检索 | |
| 1课时 | **开发环境与工具链** | 直播 |
| | LangChain 核心概念（Chain/Retriever/Memory）；Vibe Coding 工作流演示（Cursor/Claude Code）；项目脚手架搭建 | |

**不出独立作业**，为模块 2 做技术储备。课上跟着敲通每个知识点的最小 Demo。

---

### 模块 2：RAG 智能知识库（7 课时，直播+代码审查）

围绕一个完整的 RAG 系统从零搭建，学员自选领域文档作为知识源。

| 课时数 | 课题描述 | 课程形式 |
|-------|--------|--------|
| 1课时 | **文档处理与切分** | 直播 |
| | 多格式文档加载（PDF/Markdown/网页）；文本切分策略（固定长度/语义切分）；元数据提取与管理 | |
| 1课时 | **知识库构建** | 直播 |
| | 文档向量化入库；索引构建与优化；检索策略（相似度/混合检索）；检索质量评估 | |
| 1课时 | **RAG Pipeline 搭建** | 直播 |
| | 检索 + 生成的完整流程；上下文注入与 Prompt 模板；引用溯源；对话历史管理 | |
| 1课时 | **多路召回与混合检索** | 直播 |
| | 关键词检索 + 向量检索的混合策略；多路召回合并与去重；Rerank 重排序模型；检索效果评估指标 | |
| 1课时 | **RAG 效果优化** | 直播 |
| | Chunk 策略调优；Prompt 迭代；Query 改写与扩展；常见问题诊断（幻觉、漏召回、答非所问） | |
| 1课时 | **API 服务封装** | 直播 |
| | FastAPI 接口设计；请求/响应模型；流式返回；错误处理；接口文档 | |
| 1课时 | **阶段联调与审查** | 直播+代码审查 |
| | 完整 RAG 系统联调；Vibe Coding 补全缺失功能；代码审查：学员讲解系统架构与关键代码 | |

**项目作业**——交付一个可运行的 RAG 知识库系统（输入问题 → 检索相关文档 → 生成带引用的回答）

---

### 模块 3：Agent 工作流进阶（6 课时，直播+代码审查）

在已有的 RAG 知识库基础上，加入 Agent 能力——让系统从「问答」升级为「能执行多步任务的智能助手」。

| 课时数 | 课题描述 | 课程形式 |
|-------|--------|--------|
| 1课时 | **Agent 核心概念** | 直播 |
| | Agent 架构（感知-决策-行动循环）；工具调用机制（Function Calling）；ReAct 推理框架 | |
| 1课时 | **工具开发与注册** | 直播 |
| | 自定义工具封装（知识库检索/网页抓取/数据处理/文件生成）；工具描述与参数设计；工具调试方法 | |
| 1课时 | **多步推理与工作流编排** | 直播 |
| | 任务分解策略；多轮工具调用；条件分支与循环；LangChain Agent / Dify 工作流实现 | |
| 1课时 | **多 Agent 协作** | 直播 |
| | 多 Agent 分工模式（调研 Agent + 写作 Agent + 审核 Agent）；Agent 间通信与任务传递；协作流程设计与调试 | |
| 1课时 | **Agent 系统集成** | 直播 |
| | 将 RAG 知识库接入 Agent 作为核心工具；对话记忆管理；异常处理与兜底策略 | |
| 1课时 | **系统联调与审查** | 直播+代码审查 |
| | 完整 Agent 系统联调；端到端测试；Vibe Coding 完善功能；代码审查：学员演示 Agent 执行多步任务 | |

**项目作业**——交付升级后的智能 Agent 系统（能调用知识库 + 多种工具，完成多步骤任务并生成结果报告）
