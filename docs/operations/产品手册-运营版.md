# Agent Hunt 产品手册 · 运营版

> 面向运营 / 业务同学，帮你理解这个网站做什么、能拿来回答什么问题、用在哪些场景，以及容易误读的地方。

---

## 一、一句话定位

**Agent Hunt 是「内部叙事手册 + 数据自检工具」，不是一个面向终端用户的产品**。

- 它**不卖课**、**不接诊断请求**、**不收集用户信息**，没有漏斗、没有激活码、没有客服动作。
- 它的用户是**你**（运营 / 业务 / 销售 / 内容同学）：让你在讲市场、招生、面试候选人、写自媒体内容时，**有数据底气**而不是话术。
- 直接面向求职者的产品在隔壁项目 [aijobfit](https://aijobfit.llmxfactor.cloud)（Agent Hunt 给它当数据生产端），那个才是有漏斗 + 加微信 + 激活码的诊断产品。

**线上地址**：<https://agent-hunt.pages.dev>

**数据规模（2026-04-30）**：9,287 条 JD（8,550 条已 LLM 标注）· 国内 + 海外双市场 · 1,392 条真实薪酬爆料（levels.fyi）· 718 条求职者画像（nowcoder）

## 二、这个网站不是什么 · 是什么

### 不是什么

| 误以为它是 | 实际不是 |
|---|---|
| 给求职者用的诊断工具 | ❌ 不是。有诊断需求请引导到 aijobfit。 |
| 实时更新的 JD 搜索引擎 | ❌ 不是。数据每周一次 export，不是实时。 |
| 课程 / 服务的销售页 | ❌ 不是。没有任何商业化功能，**全开放 / 全免费 / 不需登录**。 |
| 招聘平台 | ❌ 不是。看的是市场结构，不是单条岗位详情。 |

### 是什么

**三种用途**：

1. **业务讲解工具** —— 招生直播 / 招生海报 / 内容分发时，把"5 条市场判断"+ "27 角色画像" 当弹药库使用。每张图都能直接截图发出。
2. **数据自检** —— 写自媒体内容、做 PPT、回答用户质疑时，回到这个网站找数字、找数据来源、找具体 JD 例子作为证据链。
3. **下游 aijobfit 的数据后台** —— 30+ 个 JSON 静态文件被 aijobfit 远程 fetch，aijobfit 报告页里所有数字都来自这里。

## 三、数据来源 · 数据怎么来的

总共 9,287 条 JD，按来源分三类：

| 来源 | 数量 | 说明 |
|---|---|---|
| **招聘平台采集** | 3,083 | Boss / 猎聘 / 拉勾 / LinkedIn / Indeed / Glassdoor。Boss 拉勾要 cookie，LinkedIn 用 JobSpy。 |
| **LLM 厂商官方 ATS** | 3,114 | OpenAI / Anthropic / xAI / Cohere / DeepMind / 智谱 / MiniMax / Moonshot / 百川 / 腾讯（1,049 条）。直连官方招聘 JSON API，没有反爬，最权威。 |
| **社区招聘帖** | 3,486 | Hacker News "Who is Hiring" + GitHub `hiring` 仓库。隐藏渠道，不进招聘平台。 |
| **真实薪酬爆料** | 1,392 | levels.fyi 上的真实到手薪酬（独立于 JD 的开价）。 |

**关键事实**：

- **所有数字都基于真实 JD 文本 + LLM 解析**。看到「医疗 + AI 增强 30k」这种数字，背后是从真实 JD 描述里抽出来的，不是行业报告口径。
- **数据每周日 02:00 UTC 自动 export**（GitHub Actions），所以站上看到的每个数字最新滞后 ≤ 7 天。
- **数据库在 Supabase ap-southeast-2**，前端 Cloudflare Pages，全 SSG 静态部署。

### 用户问"数据准吗"怎么答

> 「9,287 条真实 JD 都是机器抓的原始文本，不是问卷或行业报告。每条 JD 用 LLM 解析出技能 / 行业 / 薪资 / 经验等字段后聚合。海外岗薪资按各币种汇率换算到人民币月薪口径（关键防呆，详见 P4 论断）。所有页面底部都标着样本量，样本 < 30 条的薪资数字仅供参考。」

## 四、三轨架构总览

首页打开看到三张并列的卡片，对应三种使用场景：

| 入口 | 路径 | 适合谁 / 何时用 |
|---|---|---|
| 📖 **叙事手册** | `/narrative` | **业务讲解 / 招生展示**。5 条经过提炼的市场判断，每条带数字 + 数据图 + 真实 JD 例子 + 业务话术 + 反例提醒。出口最高的就是这个轨。 |

## 五、5 条市场判断 · 逐条解读

每条都在 `/narrative/p{1-5}` 有详情页（含图表 + 真实 JD 例子 + 业务话术）。下面是核心要点 + 业务用法。

### P1 · 市场基本盘 —— 国内传统行业 AI 增强需求 = 互联网的 3.4×

**关键数字**：医疗 + 制造 + 汽车 + 金融 + 教育 五个传统行业的 AI 增强 JD 加起来是 **685 条**，互联网行业 AI 增强 JD 是 **201 条**，比值 **3.41**。

**这个数字怎么用**：

- **招生话术**：「程序员转 AI 程序员的赛道每个机构都在挤；传统行业用 AI 这条赛道是互联网的 3.4 倍大，但市面零课程覆盖。」
- **新人讲解**：「你不必转码，传统行业要 AI 落地的人比互联网多得多。」
- **配套图表**：行业分布柱状图 + 5 条真实传统行业 AI 增强 JD 例子（医疗 / 制造 / 汽车 / 金融 / 教育各 1）。

**反例提醒（用户可能质疑）**：

- 「3.4× 是不是把零售 / 通信全算进去了？」 → 不是。只算了 5 个传统行业。完整行业分布表在 `/narrative/p1`。
- 「685 条 vs 201 条样本是不是太小？」 → 这是国内 AI 增强 JD 的全样本（已剔除 ai_native）。基数不大但比例稳定。
- 「互联网公司也在做 AI 增强啊？」 → 互联网公司做 AI native 的 PM / 算法岗已经被纳入 ai_native 类别（约 2316 条），AI 增强是指岗位本身不是 AI 但要求 AI 工具能力。

### P2 · 薪资反直觉 —— 医疗 / 制造 / 金融 AI 增强岗，比互联网高 20%

**关键数字**：

- 高薪传统行业（金融 / 医疗 / 制造）AI 增强中位月薪 **30,000**（n=290）
- 互联网 AI 增强中位月薪 **25,000**（n=191）
- 溢价 **+20%**

**这个数字怎么用**：

- **打破学员心智**：「学员一来都觉得互联网 = 高薪。但 AI 增强这个细分赛道里，互联网 25k vs 金融/医疗/制造 30k，传统行业反向溢价。」
- **配套图表**：行业 × 薪资 P25/P50/P75 横条图（每行有样本量）+ 高薪传统 vs 互联网真实 JD 对比例子。

**反例提醒**：

- 「能源 48k 那条是不是只有几条 JD 撑起来的？」 → 是。能源 n=11，样本太小，只展示中位作参考，**不要在话术里把能源 48k 单独挑出来当卖点**。话术只用「金融 / 医疗 / 制造」三个 n>50 的行业。
- 「20% 溢价是中位口径，分位呢？」 → P25/P75 在详情页，溢价稳定（P25 +13% / P75 +25%）。

### P3 · 新岗位品类 —— OpenAI / Anthropic 工程师里 ~17% 是「桥梁工程师」

**关键数字**：

- 5 大 LLM 厂（OpenAI / Anthropic / xAI / Cohere / DeepMind）共 1,009 条工程岗位
- 其中 **171 条** 属于 Forward Deployed / Solutions / Applied / Implementation Engineer（客户对接 + 落地工程）
- 占比 **17%**（OpenAI 16.9% / Anthropic 13.5%）

**这个数字怎么用**：

- **新品类介绍**：「LLM 厂商需要的不是写算法的人，是把模型落地到客户业务的人——这种岗位国内课程市场零覆盖，门槛 = 工程能力 + 客户沟通 + 技术演讲三合一。」
- **海外定位**：作为「想出海 / 想做海外 remote」学员的具体目标方向。
- **配套图表**：5 家 LLM 厂客户对接工程师占比柱状图 + 真实 OpenAI / Anthropic 官方 ATS 抓取的 JD 例子。

**反例提醒**：

- 「国内有这种岗吗？」 → 国内的「Solutions Engineer / Applied AI」目前主要在出海 SaaS 团队（飞书 / 钉钉 / 火山等部分海外业务），数量远少于美股 LLM 厂。学员想做这条线**目前主要靠出海 / 海外 remote**。
- 「这岗薪资如何？」 → 美股 LLM 厂 base 通常 $180k-$300k + 股权，levels.fyi 数据在 `/salary` 页可查。

### P4 · 跨市场套利 —— 海外 AI 增强是国内的 2.78×

**关键数字（汇率换算后）**：

- AI 原生：国内中位 32.5k vs 海外 78.8k，**ratio 2.43×**
- AI 增强：国内中位 23.0k vs 海外 63.9k，**ratio 2.78×**

**这个数字怎么用**：

- **出海赛道定位**：「海外做 AI 增强的 ratio 略高于做 AI 原生算法。这个洞察直接挑战『出海要会算法』的常见误判。」
- **打破网传错误数字**：「网上流传的『海外是国内 4-5 倍』是没换算汇率的口径错（USD/年 直接当 CNY/月 比）。修正后实际 2.4-2.8 倍。」

**反例提醒（关键，必读）**：

- ⚠️ **必须用汇率换算后口径**。Job 模型里有 `salary_mid_cny_monthly` 防呆属性，所有跨市场聚合都自动换算。
- 「2.78× 是真实购买力差距吗？」 → 不是。2.78× 是名义薪资差。扣掉海外旧金山 / 纽约的生活成本 + 签证 / 远程摩擦，**真实购买力差距约 1.5-2×**，不是 2.78×。详情页有这条 caveat。
- 想强调高 ratio 时用 P4；想强调"出海没有传说中那么夸张"时也用 P4，关键看场景。

### P5 · 预期管理 —— 海外幽灵岗集中度是国内 3×

**关键数字**：

- 海外「同公司 + 同标题」重复发 ≥ 5 次的簇 30 个，占 5,502 条 JD 的 **0.55%**
- 国内同标准 7 个簇，占 3,795 条的 **0.18%**
- 比值 **3.1×**

**这个数字怎么用**：

- **预期管理**：「学员看 LinkedIn 上 5,000+ active jobs 不要兴奋。海外岗位重复发布 + 没真意向的"幽灵岗"远多于国内，建议先打 30% 折扣再算实际可投岗。」
- **典型例子**：Deloitte 一个 Full Stack Engineer 标题挂 19 次，Meta 同样 PM 标题 17 次。
- **配套表格**：Top 10 幽灵岗（公司 / 标题 / 重复次数 / 市场 / 平台）。

**反例提醒**：

- 「19 次 ≠ 完全是 hiring 假象」 → 严格说"同公司同标题重发"是嫌疑信号，不能 100% 等同于幽灵岗。但 19 次、17 次这种极端值，几乎可以确认是单一岗位反复发布。话术里说「**疑似幽灵岗**」更稳妥。
- 0.55% / 0.18% 是「严判幽灵簇 / 总 JD」的比，分母是全部 JD，所以绝对占比看起来低。**真实"水分"≠ 0.55%**，这只是裸"重复名"信号；考虑到岗位实际撒网海投行为，实际能投的还要再打折。

### 怎么用

- **面试候选人**：候选人说"我想做 algorithm"时，把详情页的"和最相邻角色的差别"指给他看（algorithm vs ai_engineer：用模型 vs 调模型）。
- **薪资底气**：发"AI 产品经理 35k"这种数字前，先翻这个表确认。
- **避免说错话**：JD 数 < 30 的角色（智能客服 / 医疗 AI / Finance Ops 等）薪资中位仅供参考，话术里**不要单独把这些角色的薪资中位拎出来当卖点**。

### 容易误用的点

- ⚠️ **「Other / 其他」混合簇不要单独引用**。国内 1,347 条、海外 1,443 条都是没分到明确簇的杂项岗位（含跨界 AI / 漫剧制作 / 传统岗带 AI 标签 / 垂直领域专家），不能当作一个角色解读。
- ⚠️ **国内 algorithm 52.5k 中位 ≠ 国内整体 AI 中位 27.5k**，容易混。前者是算法工程师，后者是国内全部 AI native 岗。讲话术前确认你说的是哪个口径。
- ⚠️ **海外 Intern/New Grad 26.4k 是中位月薪**，不要当年薪。海外校招初级岗位月薪折算后看起来低，但 base + 股权打满是另一回事。

## 八、典型使用场景

### 场景 1 · 招生直播 / 短视频

- **用法**：把 `/narrative/p1` 到 `/narrative/p5` 的 5 张大数字截图，做成 5 张幻灯片当开场。每张配 60 字业务话术（详情页有「一句话讲述」可直接复制）。
- **避坑**：不要把 P3 桥梁工程师的薪资描述成"人人都能去"。门槛是工程能力 + 英文沟通 + 客户演讲三合一，至少 5 年工程经验起步。

### 场景 2 · 1V1 学员定位

- **用法**：让学员在 `/roles` 翻角色卡，3 分钟内圈出 2-3 个感兴趣方向，然后逐个看详情页的"适合谁 / 不适合谁 / 和邻居角色差别"。
- **避坑**：学员说"我想做算法工程师"时，**先看 `/roles/domestic/algorithm` 的"不适合谁"那段**，让他自己判断。

### 场景 3 · 自媒体内容

- **用法**：每条自媒体内容对应一条叙事手册的论断（已经做过映射，详情见 [`content/`](../../content/) 目录）。
- **避坑**：发数字前先回 `/narrative/p{x}` 确认最新口径——数据是周更的，3 周前抓的数字今天可能略有变化。

### 场景 4 · 用户质疑兜底

- 用户质疑数字 → 让他看 `/narrative/p{x}` 的「数据来源」段（页面底部）。
- 用户质疑"准吗" → 让他看 `/report` 顶部的全样本说明 + 9,287 条 JD 标注情况。
- 用户质疑"海外比国内高 4-5 倍" → P4，告诉他网传是没换汇率的口径错。

### 场景 5 · 数据自检

- 写 PPT / 公众号要引用一个数字时，**先在站内复现这个数字**，确认页面在最新数据下还能看到。
- 数字找不到了 → 大概率上一周的导出脚本里口径变了（比如 `/narrative/p4` 的 ratio 在 v0.9 修了汇率 bug 从 4.6× 改成 2.78×），用最新口径，**不要用旧记忆**。

## 九、数据更新机制

| 时间 | 动作 | 触发 |
|---|---|---|
| 每周日 02:00 UTC | 自动跑 export 脚本 + 重新部署到 Cloudflare | GitHub Actions cron |
| 任意时间手动触发 | 重新跑数据采集（HN / GitHub / 腾讯 / 全部） | GitHub Actions workflow `collect-data.yml` |
| 任意时间手动触发 | 单独 build + deploy 前端（不重抓数据） | `npm run build && npx wrangler pages deploy out` |

**对运营的意义**：

- 你今天看到的数字 = 上个周日 02:00 UTC 跑完 export 后的快照。
- 不会出现"昨天加的数据今天还没显示"——除非工程同学手动跑了 collect-data。
- 写文案前可以先看 footer 的 export 时间戳（如果暴露的话）确认数据新鲜度。

## 十、常见误用 / 误读

❌ **把 Agent Hunt 当 aijobfit** —— Agent Hunt 是后台/数据，aijobfit 才是给求职者用的诊断产品。学员说"我想测匹配度"时引到 aijobfit，不是这里。

❌ **把"其他/Other"混合簇当一个角色** —— 国内 1,347 条、海外 1,443 条 other 是没归类的杂项，不能讲"其他岗位中位 20k"。话术只用 14 个国内 + 11 个海外有明确簇的角色。

❌ **把 P4 ratio 用没换汇率的版本** —— 任何"海外比国内高 X 倍"的话术都必须引用站上 2.43× / 2.78× 的版本。引用 4-5× 是错。

❌ **拿小样本行业 / 角色的薪资中位当卖点** —— 能源 48k（n=11）、医疗 AI 10k（n=14）、智能客服 10.5k（n=5）这种 n<30 的数字只能讲"作为参考"，不能讲"行业薪资就是 X"。

❌ **"看 LinkedIn 5,000 个 jobs 我能投 5,000 个"** —— P5 提示 30% 折扣（实际更狠），用幽灵岗 + 同岗位反复发 + 海投实际响应率三个角度复盘。

❌ **国内 + 海外混着算薪资** —— **绝对禁止**。国内 / 海外**必须分别独立分析**（这是产品的核心约束，是 memory 里写死的规则）。任何"国内国外平均薪资 X"的话术都是错的。

❌ **公开质疑"为什么腾讯有 1049 条"** —— 因为腾讯 careers.tencent.com 有公开 JSON API 而且关键词矩阵覆盖广，不是数据偏向。其他大厂（字节 / 阿里 / 百度 / 商汤 / 阶跃 / 零一万物）SPA 反爬重，目前没采到，未来会补。
