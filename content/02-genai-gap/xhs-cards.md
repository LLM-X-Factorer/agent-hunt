## Top 1 就不一样

国内 AI 岗位 JD 里出现率最高的技能。

**LLM（6.1%）**

海外呢？

**Python（4.2%）**

第一名都不是同一种东西。

国内招的是会用大模型的人，海外招的是能建 AI 系统的人。

## Top 5 对照

国内前 5：

LLM (6.1%) → Agent Architecture (2.4%) → Python (1.6%) → RAG (1.0%) → Prompt Engineering (0.9%)

海外前 5：

Python (4.2%) → LLM (2.5%) → Generative AI (2.3%) → SQL (1.8%) → Machine Learning (1.7%)

国内全是「调模型的工具」。

海外是「工程基础 + AI 概念」。

## 更扎心的是 Top 10

海外 top 10 里有 Python / SQL / ML / AWS / PyTorch / GenAI。

国内 top 10 里：除了 Python 排第 3（1.6%），SQL / ML / AWS / PyTorch / GenAI **一个都没进**。

不是国内程序员不会。

是国内 JD 把工程基础当作「隐含技能」，只写 AI 关键词。

## Python 的故事

海外 4.2% 的 AI 岗明确要求 Python，国内只有 1.6%。

但你能找到一个不写 Python 的国内 AI 工程师吗？

海外 JD 必须写明，因为他们要区分会 Python 的和只会 Prompt 的。

写不写 Python，决定了你筛进的是哪种岗位。

## 想去海外？把这 5 样写到简历最前面

**Python** — 生产级工程代码，不是写脚本。

**SQL + 数据分析** — 很多 AI 岗本质是数据 + 模型混合岗。

**ML 基础** — 不只是调 API，是训练 / 评估 / 调优思路。

**至少一个云** — AWS 优先，GCP 次之。

**Generative AI / RAG 实战经验** — 海外这条线和应用层是同步的。

国内 JD 不写的东西，恰好是海外 JD 必写的。
