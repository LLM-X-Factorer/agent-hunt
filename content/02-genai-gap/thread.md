# X Thread：国内和国际 AI 招聘在说两种语言

---

## Thread（7 条）

### 1/7（钩子）

国内天天喊 AIGC，但你知道"Generative AI"这个词在国内 AI 岗位 JD 里出现的比例是多少吗？

0.2%。

国际是 21.5%。差了 107 倍。

但这还不是最离谱的。🧵

### 2/7（全景数据）

我分析了 1213 个国内 + 331 个国际 AI 岗位的 JD，发现技能需求差距最大的 Top 5：

- Python：国内 3.1% vs 国际 40.8%（差 37.7%）
- Generative AI：0.2% vs 21.5%（差 21.3%）
- AWS：0.2% vs 18.1%（差 17.9%）
- Machine Learning：0.7% vs 18.1%（差 17.4%）
- MLOps：0% vs 8.5%（国内完全空白）

### 3/7（解读：两种语言）

国内 JD 的高频词：LLM、Agent、RAG、大模型
国际 JD 的高频词：Python、ML、Generative AI、AWS、SQL

国内在招"会用大模型的人"。
国际在招"能建 AI 系统的人"。

一个在找应用层，一个在找工程层。这不是差距，是两个世界。

### 4/7（Python 的故事）

Python 是最荒谬的例子。

国际 40.8% 的 AI 岗位要求 Python。国内只有 3.1%。

不是国内 AI 工程师不写 Python——他们写。但国内 JD 不提。因为国内 JD 只写"大模型开发经验"，把 Python 当作"你肯定会的"来处理。

国际的逻辑是：我得写清楚，因为我要区分会 Python 的和只会 Prompt 的。

### 5/7（MLOps = 0 的含义）

MLOps 在国内需求是 0。不是少，是 0。国际 8.5%。

MLOps 是什么？模型的部署、监控、版本管理、A/B 测试、持续训练。

国内不需要 MLOps 不是因为我们太先进了跳过了，是因为大部分国内公司的 AI 还没到需要 MLOps 的阶段——还在 demo 阶段，没进生产。

### 6/7（这意味着什么）

如果你是国内 AI 工程师，想去国际市场：

1. Python 工程化能力（不是会写脚本，是会写生产级代码）
2. 云平台（AWS/Azure/GCP 至少会一个）
3. ML 基础（不只是调 API，理解模型训练和评估）
4. MLOps（最容易被忽视，也最稀缺）

这四样东西，国内 JD 几乎不提。但国际市场天天在找。
