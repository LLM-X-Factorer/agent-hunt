# X Thread：国内和国际 AI 招聘在说两种语言

> 数据基线：v0.11（2026-05）—— 22 个数据源 9,287 条 JD，国内 3,786 / 海外 5,501

---

## Thread（7 条）

### 1/7（钩子）

国内 AI 岗位 JD 里出现率最高的技能：LLM（6.1%）。
海外 AI 岗位 JD 里出现率最高的技能：Python（4.2%）。

第一名就不是同一种东西。

我分析了 9,287 条 AI 岗位 JD，发现国内外不是「水平差距」，是「在招两种人」。🧵

### 2/7（top 5 对照）

**国内 top 5**：LLM (6.1%) → Agent Architecture (2.4%) → Python (1.6%) → RAG (1.0%) → Prompt Engineering (0.9%)

**海外 top 5**：Python (4.2%) → LLM (2.5%) → Generative AI (2.3%) → SQL (1.8%) → Machine Learning (1.7%)

国内的前 5 全是「调模型的工具」。
海外前 5 是「工程基础 + AI 概念」。

[附图 1：top 10 双列对照]

### 3/7（结构性的差距）

更扎心的是 top 10：

海外 top 10 里有：Python、SQL、ML、AWS、PyTorch、GenAI——工程基础 + AI 概念都齐了。

国内 top 10 里：除了 Python 排第 3（1.6%），SQL / ML / AWS / PyTorch / GenAI **一个都没进**。

不是「国内不会」，是国内 JD 不写。

### 4/7（这意味着什么）

国内招"会用大模型的人"。
海外招"能建 AI 系统的人"。

一个在找应用层（LLM 调用 + Agent 编排 + Prompt 调优）。
一个在找工程层（Python 写生产代码 + SQL 取数 + 云上部署 + 训练循环）。

不是差距，是两个世界的招聘语言。

### 5/7（Python 的故事）

Python 是这件事最直白的证据。

海外 4.2% 的 AI 岗位明确要求 Python，国内只有 1.6%——但你能找到一个不写 Python 的国内 AI 工程师吗？

国内 JD 把 Python 当"隐含技能"，只写 AI 关键词。海外 JD 必须写明，因为他们要区分会 Python 的和只会 Prompt 的。

写不写 Python，决定了你筛进的是哪种岗位。

### 6/7（如果你要去海外）

国内 AI 工程师想转海外市场，简历要把这些写到最前面（按海外排序）：

1. Python（不是脚本，是生产级工程代码）
2. SQL + 数据分析（很多 AI 岗位本质是数据 + 模型混合岗）
3. ML 基础（不只是 API，是训练 / 评估 / 调优思路）
4. 至少一个云（AWS 优先，GCP 次之）
5. Generative AI / RAG 经验（这是同步的）

国内 JD 不写的东西，恰好是海外 JD 必写的。简历里不写，等于你给海外 HR 看了一份只剩 LLM 关键词的简历。

### 7/7（收尾 + 引流）
