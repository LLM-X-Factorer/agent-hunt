# 公众号长文：国内和国际 AI 招聘，在说两种完全不同的语言

## 标题

「9287 条 JD 告诉你：国内 AI 招的是『会用大模型的人』，海外招的是『能建 AI 系统的人』」

## 正文

---

### 一、第一名就不一样

我从 22 个数据源采集了 9,287 条 AI 岗位 JD（国内 3,786 / 海外 5,501），用大模型做结构化解析后做技能词频统计。

国内 AI 岗位 top 1 是 **LLM（6.1%）**。

海外 AI 岗位 top 1 是 **Python（4.2%）**。

9,287 条 JD 数据——**国内招「会用 AI 的人」，海外招「能建 AI 系统的人」。**

[图 1：国内/海外 top 1 单点冲击]

你可能会说，这只是 top 1 的差异。但当我把 top 5、top 10 都拉出来看，发现这不是个例——**国内和海外的 AI 招聘，从头到尾都在说两种完全不同的语言。**

### 二、两张完全不同的技能清单

国内 AI 岗位 top 5 技能：

1. LLM（6.1%）
2. Agent Architecture（2.4%）
3. Python（1.6%）
4. RAG（1.0%）
5. Prompt Engineering（0.9%）

海外 AI 岗位 top 5 技能：

1. Python（4.2%）
2. LLM（2.5%）
3. Generative AI（2.3%）
4. SQL（1.8%）
5. Machine Learning（1.7%）

[图 2：top 10 双列对照]

看到区别了吗？

**国内的关键词是：LLM、Agent、RAG、Prompt。** 围绕"调模型的工具"展开。

**海外的关键词是：Python、SQL、ML、GenAI、AWS。** 围绕"建 AI 系统的基础"展开。

一个在招会用大模型的人。一个在招能建 AI 系统的人。

### 三、更扎心的是 top 10

把视野放到 top 10，结构差异更明显。

海外 top 10 里有：Python、LLM、GenAI、SQL、ML、AWS、Prompt Engineering、Data Analysis、PyTorch、Java——工程基础（Python / SQL / AWS / PyTorch）+ AI 概念（LLM / GenAI / ML）都齐了。

国内 top 10 里：LLM、Agent Architecture、Python、RAG、Prompt Engineering、Java、Data Analysis、LangChain、ML、SQL——除了 Python 排第 3（1.6%），SQL / ML 各只有 0.5% 排在末尾，AWS / PyTorch / GenAI **一个都没进**。

不是国内 AI 工程师不会 SQL、不懂 AWS。问题在于**国内 JD 把这些当成"隐含技能"，写都不写**。

### 四、Python 的故事

Python 是这件事最直白的证据。

海外 4.2% 的 AI 岗位明确要求 Python，国内只有 1.6%。

但你能找到一个不写 Python 的国内 AI 工程师吗？

国内 JD 把 Python 当作"默认技能"——"你来面 AI 岗，你肯定会 Python 吧？"

海外不是这样。海外 JD 明确写出 Python，是为了区分两类人：**能用 Python 写生产级代码的人**和**只会写 Prompt 的人**。

国内 JD 不做这个区分。这意味着国内市场还没有感受到"会 Prompt ≠ 会工程"的痛。

但这个痛会来的——当 AI 应用从 demo 走向生产、需要持续运维和迭代的时候。

### 五、为什么会这样

这不是"国内落后"这么简单。两个市场的差异有结构性原因：

**1. 国内 AI 招聘是被 LLM 浪潮拉起来的。** ChatGPT 之后，国内大量涌入的 AI 岗位是围绕"用大模型做应用"展开的——接 API、搭 Agent、做 RAG。这些工作的入门门槛在「调大模型」上，不在传统 ML 功底上。

**2. 海外 AI 市场已经经历了完整的 ML 时代。** 在 LLM 之前，海外已经有大量成熟的 ML 岗位、Data Engineering 岗位、ML Infrastructure 岗位。LLM 是叠加在这个基础上的新层，不是替代。所以海外 JD 会同时要求 Python + SQL + ML + LLM。

**3. 国内 JD 写作风格本身偏简洁。** 国内招聘文化里，技术 JD 普遍只列"差异化要求"，把通用技能（Python / SQL / Linux）当成隐含。海外招聘文化里，JD 是法律意义上的「岗位说明」，必须写明所有要求。

### 六、对你意味着什么

**如果你只在国内市场求职：** 短期内 LLM + Agent + RAG + Prompt 够用了。但随着行业成熟，工程化能力（Python 生产级代码、云部署、ML 基础）的需求一定会上升——这些是「国内 JD 里看不到、海外 JD 里必写」的能力。

**如果你想进入海外市场：** 你的简历可能需要重新校准。把这 5 样写到最前面：

1. **Python（生产级工程代码）** — 不是写脚本，是写可测试、可部署、有 CI/CD 的代码
2. **SQL + 数据分析** — 很多 AI 岗本质是数据 + 模型混合岗，不只是接 API
3. **ML 基础** — 不只是调 API，是训练 / 评估 / 调优思路
4. **至少一个云（AWS 优先）** — 海外大量 AI 岗位有云原生要求
5. **Generative AI / RAG 实战经验** — 这条线和国内是同步的，不要砍掉

国内 JD 不写的东西，恰好是海外 JD 必写的。简历里不写，等于你给海外 HR 看了一份只剩 LLM 关键词的简历。

**如果你是团队负责人：** 问自己一个问题——你的 AI 产品是 demo 还是生产系统？如果是后者，你可能很快就需要上面这些"国内 JD 里看不到"的技能了。
