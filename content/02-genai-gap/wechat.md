# 公众号长文：国内和国际 AI 招聘，在说两种完全不同的语言

## 标题

「1500 条 JD 告诉你：国内和国际 AI 招聘，在说两种完全不同的语言」

## 正文

---

### 一、一个荒谬的数字

"Generative AI"——生成式 AI。

这个词你每天都在刷到。国内每篇 AI 文章都在讲 AIGC、大模型、生成式 AI。

但我从 1213 个国内 AI 岗位的 JD 里搜"Generative AI"这个技能要求，只找到了 2 个。

占比 0.2%。

国际市场呢？331 个岗位里有 71 个要求这个技能。占比 21.5%。

**差了 107 倍。**

[图：Generative AI 0.2% vs 21.5%]

你可能会说，这只是一个词的差异。但当我把所有技能的差距拉出来看的时候，发现这不是个例——**国内和国际的 AI 招聘，从头到尾都在说两种完全不同的语言。**

### 二、两张完全不同的技能清单

国内 AI 岗位 JD 最常提到的技能：

1. LLM（14.3%）
2. Agent Architecture（5.8%）
3. Python（3.1%）
4. RAG（1.8%）
5. Prompt Engineering（1.6%）

国际 AI 岗位 JD 最常提到的技能：

1. Python（40.8%）
2. LLM（23.9%）
3. Generative AI（21.5%）
4. SQL（19.0%）
5. Machine Learning（18.1%）

[图：技能差距 Top 10]

看到区别了吗？

**国内的关键词是：LLM、Agent、RAG。** 围绕"大模型应用"展开。
**国际的关键词是：Python、ML、SQL、云平台。** 围绕"AI 工程化"展开。

一个在招会用大模型的人。一个在招能建 AI 系统的人。

### 三、三个最触目惊心的差距

#### 1. Python：3.1% vs 40.8%

这是差距最大的一个，37.7 个百分点。

不是国内 AI 工程师不写 Python。他们当然写。但国内 JD 不提 Python，因为它被当作"默认技能"——"你来面 AI 岗，你肯定会 Python 吧？"

国际不是这样。国际 JD 明确写出 Python，是为了区分两类人：**能用 Python 写生产级代码的人**和**只会写 Prompt 的人**。

国内不做这个区分。这意味着国内市场还没有感受到"会 Prompt ≠ 会工程"的痛。但这个痛会来的。

#### 2. 云平台（AWS/Azure/GCP）：≈ 0% vs 15-18%

国际市场 18.1% 的 AI 岗位要求 AWS，16% 要求 Azure，14.8% 要求 GCP。而且有趣的是，这三个经常同时出现——企业采用"多云策略"。

国内呢？每个都只有 0.2%。约等于零。

这说明什么？国内的 AI 应用大部分还跑在本地或私有集群上。国际已经在用云原生的方式部署 AI——弹性扩缩、多区域部署、成本优化。

这不是技术选择的差异，是**成熟度的差异**。

#### 3. MLOps：0% vs 8.5%

MLOps 在国内的需求是 0。不是少，是 0。

MLOps 做什么？模型版本管理、自动化训练流水线、A/B 测试、监控漂移、持续部署。

国内不需要 MLOps，不是因为我们跳过了这个阶段。是因为大部分国内公司的 AI 产品还在 demo 和 POC 阶段。Demo 不需要 MLOps。上了生产、有了真实用户、需要持续迭代的时候，才需要。

**MLOps = 0 这个数字，比任何分析都清楚地说明了国内 AI 产业化的真实进度。**

### 四、为什么会这样

这不是"国内落后"这么简单。两个市场的差异有结构性原因：

**1. 国内 AI 爆发是由 LLM 驱动的。** ChatGPT 之后，国内大量涌入的 AI 岗位是围绕"用大模型做应用"展开的——接 API、搭 Agent、做 RAG。这些工作不太需要传统 ML 功底，也不太需要云平台。

**2. 国际 AI 市场已经经历了 ML 时代。** 在 LLM 之前，国际市场已经有大量成熟的 ML 岗位。LLM 是在这个基础上叠加的，不是替代的。所以国际 JD 会同时要求 Python + ML + LLM + 云平台。

**3. 国内的"AI 落地"还在早期。** 真正把 AI 放进生产系统、影响业务指标、需要运维和迭代的公司还是少数。大部分还在"做出来看看效果"的阶段。

### 五、对你意味着什么

**如果你只在国内市场求职：** 短期内 LLM + Agent + RAG 够用了。但随着行业成熟，工程化能力（Python 生产级代码、云部署、MLOps）的需求一定会上升。

**如果你想进入国际市场：** 你的技能栈可能需要重新校准。补四样东西：
1. **Python 工程化** — 不是写脚本，是写可测试、可部署、有 CI/CD 的生产代码
2. **云平台** — AWS/Azure/GCP 至少精通一个
3. **ML 基础** — 不只是调 API，理解模型训练、评估、特征工程
4. **MLOps** — 目前国内最稀缺、国际最需要的技能

**如果你是团队负责人：** 问自己一个问题——你的 AI 产品是 demo 还是生产系统？如果是后者，你可能很快就需要上面这些"国内 JD 里看不到"的技能了。

### 六、你在哪个频道上？

不同市场说不同的语言，没有对错。但你得知道自己在哪个频道上，以及你想去哪个频道。
